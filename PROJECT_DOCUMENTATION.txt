PROJECT OVERVIEW
----------------
This is a complete Learning Management System (LMS) built using pure HTML, CSS, 
and JavaScript. The system provides a modern, user-friendly interface for 
managing courses, assignments, grades, and student profiles. All data is stored 
locally in the browser using the Web Storage API (localStorage).


TECHNOLOGY STACK
----------------
- Frontend: HTML5, CSS3, JavaScript (ES6+)
- Storage: Browser localStorage API
- No external dependencies or frameworks
- Pure vanilla JavaScript implementation

DATA STRUCTURE
--------------

The application uses localStorage to store data in JSON format:

1. Users Array (key: 'users')
   Structure:
   [
     {
       id: "unique_id",
       name: "User Full Name",
       email: "user@example.com",
       password: "hashed_password",
       createdAt: "ISO_date_string"
     }
   ]

2. Current User (key: 'currentUser')
   Structure:
   {
     id: "user_id",
     name: "User Name",
     email: "user@example.com",
     password: "password",
     createdAt: "ISO_date_string"
   }

3. Courses Array (key: 'courses')
   Structure:
   [
     {
       id: "unique_id",
       name: "Course Name",
       code: "CS101",
       instructor: "Instructor Name",
       description: "Course description",
       userId: "user_id",
       createdAt: "ISO_date_string"
     }
   ]

4. Assignments Array (key: 'assignments')
   Structure:
   [
     {
       id: "unique_id",
       title: "Assignment Title",
       courseId: "course_id",
       courseName: "Course Name",
       dueDate: "YYYY-MM-DD",
       description: "Assignment description",
       userId: "user_id",
       status: "pending" | "completed",
       createdAt: "ISO_date_string"
     }
   ]

5. Grades Array (key: 'grades')
   Structure:
   [
     {
       id: "unique_id",
       userId: "user_id",
       courseId: "course_id",
       courseName: "Course Name",
       assignmentId: "assignment_id",
       assignmentTitle: "Assignment Title",
       score: 85,
       date: "ISO_date_string"
     }
   ]

SECURITY CONSIDERATIONS
-----------------------

1. Password Storage
   - Currently stored in plain text (localStorage limitation)
   - In production, passwords should be hashed
   - Consider using bcrypt or similar

2. XSS Prevention
   - escapeHtml() function prevents script injection
   - All user input is sanitized before display

3. Authentication
   - Session management via localStorage
   - Automatic redirect for unauthorized access
   - User-specific data filtering

4. Data Validation
   - Client-side form validation
   - Email format checking
   - Password length requirements
   - Required field validation


RESPONSIVE DESIGN
-----------------

The application is fully responsive with breakpoints at:
- Mobile: < 768px
  * Stacked navigation
  * Single column layouts
  * Full-width cards
  * Simplified menu

- Tablet: 768px - 1024px
  * Two-column grids
  * Sidebar remains visible

- Desktop: > 1024px
  * Full sidebar navigation
  * Multi-column grids
  * Optimal spacing


USAGE INSTRUCTIONS
------------------

   - Click "Sign Up" on login page
